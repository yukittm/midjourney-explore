# Lessons Learned

- **Archive legacy into `/archive/`, keep the live tree clean**: LLM-assisted work accumulates files; I left superseded docs in place (V7 research bundle in `docs/research/`, V7 automation idea in `automation/`) with only "ARCHIVE" banners instead of moving them, and never created an archive folder the user had asked for. → When a doc is superseded, **move it to top-level `/archive/`** (cautious deletion = *move, not delete*; preserve provenance), update `/archive/README.md` + every pointer, and return the repo to a clean current-only state. Default to pruning/relocating legacy, not letting it pile up.
- **Legacy-carryover bias on creative subjects**: When asked for new MJ prompts I repeatedly recycled prior subjects (antelope-rider, heroless cracked-earth landscapes) instead of inventing fresh ones — a preservation bias that smuggles stale intent forward. → When the user asks for new subject matter, generate genuinely NEW subjects; never reuse a specific subject/scene from earlier prompts unless explicitly told to iterate. The giz surreal-natural WORLD is the constant; the specific subjects must be fresh each time.
- **MJ prompt length: don't ship sparse, but length is NOT the lever**: I repeatedly shipped lazy/sparse prompts; then over-corrected into a dogmatic ~40-word target. In-app (2026-06-23) a ~85-word body AND the simple ~20-word originals both produced the look. → Don't default to brevity (specify the subject + the real background noun concretely), but **don't chase a word count** — the actual levers are the realism **kernel** (`--style raw` + low `--s` + photo cues + `--no painting…`) + **naming the real subject/phenomenon** (incl. the bridge) + the **`--sref`**. See `docs/style/prompting.md` §7. (Supersedes the earlier "~40-word target, count before sending" rule.)
