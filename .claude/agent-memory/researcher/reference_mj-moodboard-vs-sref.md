**Moodboard = a personalization style PROFILE built from MANY images, referenced with `--p <code>` (NOT --sref).** Official Moodboards doc (via search snippet): "Moodboards are a type of Midjourney personalization that lets you select specific images to create a unique style." You add images (upload / URL / gallery); "As you add more diverse images to the board, the model will start to remix them in more complex ways." Each moodboard has a unique ID → copy from Moodboards Page, append as `--p mID` (Discord: `--p` alone = default profile, `--p CODE` = specific). Compatible with MJ v6, v7, v8.1. **CANNOT be combined with `--sv` (style-ref version) or `--sw` (style weight)** — official. Editable/live: changing the board regenerates new codes (versions). Strength tuned via `--s` (stylize), NOT --sw.

**Personalization `--p` family — re-verified 2026-06-26 against docs.midjourney.com (403 on fetch; verbatim from search snippets). Three sibling official articles now exist: Personalization (32433330574221), Moodboards (39193335040013), Profiles (41117938447629), plus Style Creator (41308374558221).**
- **`--p` carries AESTHETIC/TASTE preference only, NOT subject/composition.** VERBATIM: "Personalization acts like a style assistant for your image creations. By liking or selecting images, Midjourney gets to know what kind of aesthetics you love and uses that information to generate images tailored to your personal tastes and unique style." Compatible MJ v6/7/8.1.
- **Invocation IDs differ by type: Profile = `--p pID`, Moodboard = `--p mID`.** VERBATIM (Personalization): "To use a specific Personalization profile, copy its ID from your Personalize page with the use icon and add it to the end of your prompt (example: --p pID)." `--p` alone = default profile(s).
- **Profile creation: pair-rating was REPLACED by grid selection.** VERBATIM (Profiles): "Rating image pairs has been replaced with selecting images from a grid. However, if you Rank Image Aesthetics from the Tasks page, this information will automatically apply to your Global Profile." Multiple profiles supported: "create extra Personalization profiles... by clicking the New Profile button"; each profile/moodboard has its own ID/code.
- **WEIGHT param for ALL personalization (profiles AND moodboards) = `--s` (Stylize), 0–1000 default 100 — NOT --sw.** VERBATIM (Personalization doc): "the stylize parameter controls how much of your Personalization is applied to the images. A lower stylize value will limit the Personalization, while a higher value will increase it." (Stylize Low=50, Med=100, High=250, V.High=750.)
- **`--sref` is the OFFICIALLY-positioned tool for cross-subject consistency** ("consistent aesthetics across multiple images... no matter the subject or scene"); Style Creator (41308374558221) makes reusable custom `--sref` codes. Moodboards = "wider aesthetic range... compared to Style References, which are more specific." `--sref` and `--p` are SEPARATE systems — VERBATIM (Style Reference): "liking style codes does not affect your Personalization profiles." Moodboard code + `--sref` code CAN be combined.

**Single-image --sref vs moodboard for carrying a bold palette across VARYING subjects — sources genuinely DISAGREE; nuance matters:**
- chasejarvis.com: moodboards average "the style of 5-10 images to create a nuanced aesthetic"; "moodboards offer the best balance of control and flexibility"; you can "remove the dark images... add brighter ones to steer the ship" — sref = "rigid consistency but lacks flexibility." → favors moodboard for palette-across-varied-subjects.
- geeky-gadgets.com: opposite framing — sref is for "a uniform color palette or design motif" / "precise stylistic replication"; moodboards "blend multiple styles, creating outputs with subtle variations" → less uniform, better for "exploratory/experimental."
- Reconciliation: a SINGLE sref image = strongest, most LITERAL transfer of THAT image's exact palette (most uniform) but most rigid. A moodboard GENERALIZES palette across N images → more robust to subject/angle variation but the palette is an average, slightly less literal. For a BOLD, SPECIFIC palette that must survive heavy subject/angle variation, a moodboard built from images that SHARE that palette is the more robust carrier; a single sref is best when you want one exact look replicated tightly. (secondary, medium confidence — official docs don't rank them.)

**Composition-imposition tradeoff — CONFIRMED that --sref does NOT impose composition (refutes the worry); moodboard generalizes:** chasejarvis: "Style References allow you to steal the aesthetic of an image without stealing its content" — focus on "lighting, color grading, and texture" not composition; IMAGE references (not sref) are "content-heavy... pull the subject matter—the shapes, the composition." Matches official sref doc (sref = style only, no composition — see [[mj-sref-image-url-behavior]]). So the premise "single sref over-imposes COMPOSITION" is WRONG for `--sref` (that's an image PROMPT). The real sref risk is over-imposing the reference's exact PALETTE/look at high --sw, not its layout. Moodboard further dilutes any single image's influence by averaging many → form varies more freely. (official + chasejarvis, high confidence on the composition point.)

**--sw (style weight) bands for "strong palette, subject still varies"** (applies to --sref only; moodboard uses --s not --sw): range 0–1000, default 100 (official). For strong-but-not-overwhelming color, the ~150–300 "strong match" band is the secondary-source rule (blakecrosley); 500+ trends "dominant," 1000 = follows ref very closely / can fight subject. midjourney-v7.com: min effective ~20; ~20 = "bright colors but more realistic," 10 = color splash only. So 100 (default)–250 is reasonable for "palette strongly present, subject still varies." (official range/default; band values secondary, medium confidence.)
